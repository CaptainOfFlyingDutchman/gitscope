# GitScope 0.2.0

GitScope 0.2.0 adds an explicit way to analyze an entire visible organization
without weakening the private allowlist default.

## What changed

- Added `gitscope analyze --all-repositories` for every organization repository visible to the configured GitHub token.
- Kept `.gitscope-repositories` as the default, least-scope workflow.
- Made `--all-repositories` and `--repos-file` mutually exclusive.
- Added resolved visible/private and contribution-selected repository counts before cloning begins.
- Extended GraphQL reserve enforcement across repository boundaries for pull requests, issues, and reviews.
- Added a cached, batched default-branch commit prefilter using configured identity emails.
- Used organization-scoped contribution searches instead of one search per visible repository.
- Limited full-history mirrors to repositories with authored pull-request, issue, review, or commit evidence.
- Limited HTML and Markdown repository tables to repositories with collected contribution activity.
- Preserved bounded Git concurrency, response caching, and offline report generation.

## Usage

The allowlist remains recommended:

```bash
gitscope analyze --org example-org --user octocat
```

To intentionally analyze every repository visible to the token:

```bash
gitscope analyze --org example-org --user octocat --all-repositories
```

All-repositories mode can substantially increase API requests and execution
time. GitScope displays the resolved scope, inspects contribution metadata, and
then reports how many repositories were selected for full-history Git analysis.
Repositories without contribution evidence are not cloned, and the configured
GraphQL safety reserve remains enforced.

Commit presence is detected on the default branch by configured author email.
Use the explicit allowlist when commits that exist only on unmerged non-default
branches, without related contribution metadata, must also be inspected.

## Installation

```bash
uv tool install \
  https://github.com/CaptainOfFlyingDutchman/gitscope/releases/download/v0.2.0/gitscope-0.2.0-py3-none-any.whl
```

Existing GitScope tool installations can upgrade with:

```bash
uv tool install --force \
  https://github.com/CaptainOfFlyingDutchman/gitscope/releases/download/v0.2.0/gitscope-0.2.0-py3-none-any.whl
```

GitScope is distributed through GitHub Releases, not PyPI. Use `SHA256SUMS` to
verify downloaded assets.
