# GitScope 0.3.0

GitScope 0.3.0 adds optional, inclusive UTC date windows while preserving
lifetime analysis as the default.

## Highlights

- Use `--since YYYY-MM-DD`, `--until YYYY-MM-DD`, or both with `gitscope analyze`.
- Commits use author dates, pull requests and issues use creation dates, and
  reviews use submission dates.
- Contribution language, file-extension, repository, timeline, and chart
  analytics now reflect the selected window.
- `--all-repositories` applies the window during commit-presence prefiltering so
  commits outside the range do not select a repository for cloning.
- Report schema 1.6 records the requested window and exposes it in offline
  HTML, Markdown, CSV, and résumé outputs.

Full bare mirrors remain intentional for repositories selected for Git
analysis. Offline exports reproduce the range already stored in `report.json`;
they do not reinterpret a lifetime report with new dates.

## Install

```bash
uv tool install \
  https://github.com/CaptainOfFlyingDutchman/gitscope/releases/download/v0.3.0/gitscope-0.3.0-py3-none-any.whl
```

For an existing installation:

```bash
uv tool install --force \
  https://github.com/CaptainOfFlyingDutchman/gitscope/releases/download/v0.3.0/gitscope-0.3.0-py3-none-any.whl
```

## Example

```bash
gitscope analyze \
  --org example-org \
  --user octocat \
  --since 2024-01-01 \
  --until 2025-12-31
```
